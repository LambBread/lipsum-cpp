# Contributing to lipsum-cpp

Thank you for considering contributing to `lipsum-cpp`!

## How to contribute

### Bug reports

If you find a bug in `lipsum-cpp`, please create a GitHub issue with a clear description and steps to reproduce.

### Feature requests

Feature requests are welcome. Please create an issue to propose any new features to `lipsum-cpp`.

### Pull requests

1. Fork the repo and create a branch from `master`.
2. Make your changes and test them adaquately.
3. Commit your changes with a clear message.
4. Push your branch to your forked repository.
5. Create a pull request explaining your changes.

### Code style

Please follow the naming conventions as described in [`src/README.md`](./src/README.md). 
Run `make pkg` before committing your changes to run `clang-format`, `clang-tidy`, and to generate version headers.

### Additional guidelines

- Add documentation where necessary.
- If applicable, add tests and/or examples for your changes.
- AI-generated pull requests are less likely to get accepted.

We look forward for your contributions!
